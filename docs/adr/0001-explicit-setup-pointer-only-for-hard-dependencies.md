# 显式 `/setup-matt-pocock-skills` 指针仅用于强依赖

工程技能依赖于由 `/setup-matt-pocock-skills` 播下的按仓库配置（问题跟踪器、分类标签词汇、领域文档布局）。有些技能在没有该配置的情况下无法有效运作——它们必须发布到特定的问题跟踪器或应用特定的标签字符串。其他的只是用它来增强输出（词汇、ADR 感知），没有它也能优雅降级。

我们将这些分为**强依赖**和**弱依赖**技能：

- **强依赖**（`to-issues`、`to-prd`、`triage`）——包含一条显式的一句话："……应该已经提供给你了——如果没有，请运行 `/setup-matt-pocock-skills`。" 没有这个映射，输出就是错的，而不仅仅是模糊的。
- **弱依赖**（`diagnose`、`tdd`、`improve-codebase-architecture`、`zoom-out`）——仅以模糊的散文引用"项目的领域词汇表"和"你所涉及领域的 ADR"。如果那些文档不存在，技能仍然有效；输出只是不那么精确。

这种划分使弱依赖技能的 token 量保持轻量，并避免将设置指针机械地复制到没有实际作用的地方。
